# Análise da Tarefa

## Objetivos da análise

- Entender o que a tarefa exige (objetivos principais).
- Ver como ela se encaixa no contexto do projeto e da funcionalidade.
- Alinhar com regras e padrões (`@.cursor/.cursor/rules`, README do projeto).
- Identificar abordagens possíveis, riscos e desafios.

## Template de resumo (preencher antes de implementar)

```
ID da Tarefa: [ID ou número]
Nome da Tarefa: [Nome ou descrição breve]
Contexto PRD: [Pontos principais do PRD que esta tarefa atende]
Requisitos Tech Spec: [Requisitos técnicos principais]
Dependências: [Lista de dependências de outras tarefas ou sistemas]
Objetivos Principais: [Objetivos primários desta tarefa]
Riscos/Desafios: [Riscos ou desafios identificados]
```

## Plano de abordagem (exemplo)

```
1. [Primeiro passo – ex: criar migration]
2. [Segundo passo – ex: criar Model e relacionamentos]
3. [Próximos passos conforme subtarefas e critérios de sucesso]
```

Ajustar os passos ao conteúdo de `[num]_task.md` e da Tech Spec.
