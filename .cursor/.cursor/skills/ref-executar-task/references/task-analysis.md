# Análise da Tarefa (Refatoração)

## Objetivos da análise

- Entender o que a tarefa exige (testes primeiro ou refatoração de código).
- Ver como ela se encaixa no PRD de refatoração (cenários de teste, requisitos técnicos).
- Alinhar com regras (`@.cursor/.cursor/rules`), README do projeto e skill `refatorar-com-testes` quando for refatorar código.
- Identificar abordagens possíveis, riscos e desafios.

## Template de resumo (preencher antes de implementar)

```
ID da Tarefa: [ID ou número]
Nome da Tarefa: [Nome ou descrição breve]
Contexto PRD: [Cenários de teste / requisitos de refatoração que esta tarefa atende]
Requisitos Tech Spec: [Se houver techspec]
Dependências: [Lista de dependências de outras tarefas ou sistemas]
Objetivos Principais: [Objetivos primários desta tarefa]
Riscos/Desafios: [Riscos ou desafios identificados]
```

## Plano de abordagem (exemplo)

```
1. [Primeiro passo – ex: criar ou estender testes para o cenário X do PRD]
2. [Segundo passo – ex: rodar testes com `npm test` ou script definido no projeto até passar]
3. [Próximos passos conforme subtarefas e critérios de sucesso]
```

Ajustar os passos ao conteúdo de `[num]_task.md`, ao PRD (cenários de teste, requisitos) e à Tech Spec se existir.
