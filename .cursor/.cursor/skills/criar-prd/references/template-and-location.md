# Template e Localização do PRD

## Template fonte

- Caminho: `./templates/prd-template.md`
- Use como estrutura exata; preencha cada seção sem pular.

## Nome do arquivo e diretório

- **Nome do arquivo:** `prd.md`
- **Diretório:** `./tasks/prd-[nome-funcionalidade]/`
- **Nome da funcionalidade:** em kebab-case (ex: `prd-checkout-pix`, `prd-relatorio-faturamento`)

## Caminho final

```
./tasks/prd-[nome-funcionalidade]/prd.md
```

## Regras ao redigir

- **O quê** e **Por quê** — não o **Como** (implementação fica na Tech Spec).
- Requisitos funcionais **numerados** para rastreabilidade.
- Documento principal com **no máximo ~1.000 palavras**; anexos se necessário.
- Seções do template: Visão Geral, Objetivos, Histórias de Usuário, Funcionalidades Principais, Experiência do Usuário, Restrições Técnicas de Alto Nível, Não-Objetivos, Questões em Aberto.
