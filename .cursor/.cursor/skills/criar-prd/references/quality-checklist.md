# Checklist de Qualidade do PRD

Validar antes de considerar o PRD pronto.

## Processo

- [ ] Perguntas de clarificação feitas e respondidas
- [ ] Plano detalhado do PRD criado (abordagem por seção)
- [ ] PRD redigido usando o template `./templates/prd-template.md`

## Conteúdo

- [ ] Requisitos funcionais numerados
- [ ] Objetivos mensuráveis
- [ ] Histórias de usuário cobrindo fluxos principais
- [ ] Não-objetivos (fora de escopo) explícitos
- [ ] Questões em aberto listadas quando houver

## Entrega

- [ ] Diretório criado: `./tasks/prd-[nome-funcionalidade]/`
- [ ] Arquivo salvo em: `./tasks/prd-[nome-funcionalidade]/prd.md`
- [ ] Caminho final informado ao usuário
- [ ] Resumo das decisões e questões abertas fornecidos

## Princípios

- Clareza sobre completude: melhor um PRD curto e claro do que longo e ambíguo.
- PRD define resultados e restrições; Tech Spec define implementação.
